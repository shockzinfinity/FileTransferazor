# Known Issues

## UI/UX

### 1. Upload 버튼 재클릭 시 파일 리스트 소실
- **현상:** Upload 버튼을 눌러 파일 선택 다이얼로그를 열었다가 취소하면, 기존 선택된 파일 리스트가 사라짐
- **원인:** `HandleSelected`에서 `loadedFiles.Clear()`를 무조건 호출
- **영향:** UI에는 이전 파일 목록이 남아있지만 실제 `loadedFiles`는 비어있어 Send 시 오류
- **해결 방향:** 파일 선택 이벤트에서 새 파일이 없으면 Clear하지 않도록 변경

### 2. 개별 파일 제거 기능 없음
- **현상:** 선택한 파일 목록에서 특정 파일만 제거할 방법이 없음
- **영향:** 잘못 선택한 파일을 빼려면 전체 재선택 필요
- **해결 방향:** 파일 목록 각 항목에 삭제 버튼 추가

### 3. 에러 메시지가 기술적
- **현상:** 업로드 에러 시 `"tus: failed to upload chunk at offset..."` 같은 raw 에러 노출
- **영향:** 사용자가 이해할 수 없는 메시지
- **해결 방향:** 에러 코드별 사용자 친화적 메시지 매핑

### 4. 에러 후 재시도 버튼 없음
- **현상:** 파일 업로드 실패 시 빨간색 바만 표시되고 개별 재시도 방법 없음
- **영향:** 전체를 다시 Send해야 함
- **해결 방향:** 실패한 파일 옆에 재시도 버튼 추가

### 5. 부분 성공 상태 불명확
- **현상:** 파일 3개 중 1개만 실패해도 전체 상태가 어떤 상태인지 불명확
- **영향:** 사용자가 성공/실패 파일을 한눈에 파악하기 어려움
- **해결 방향:** 전체 요약 표시 (예: "2/3 파일 업로드 완료, 1개 실패")

## 기능

### 6. Blazor WASM 빌드 캐시 문제 (개발 환경)
- **현상:** 코드 변경 후 빌드 시 브라우저가 이전 WASM 해시를 캐시하여 404 발생
- **영향:** 개발 중 브라우저 캐시를 수동으로 삭제해야 함 (Cmd+Shift+R 또는 Disable cache)
- **원인:** Blazor WASM의 파일 해시 기반 캐싱 메커니즘
- **비고:** 개발 환경 전용 이슈, 프로덕션 배포 시 blazor.boot.json이 자동 처리

## 향후 개선

### 8. tus 미완료 업로드 주기적 정리
- **현재:** 서버 시작 시 만료 파일 삭제 (ExpirationHours 기준)
- **개선:** Hangfire recurring job으로 주기적 정리 추가

### 9. TusDiskStore → 커스텀 S3 Store 전환
- **현재:** tus 임시 → 디스크, 완료 후 → S3/Local (2단계)
- **개선:** tus chunk를 S3 Multipart Upload Part로 직접 전송 (디스크 의존 제거)

### 10. 다중 파일 다운로드
- **현재:** 파일 단위 개별 다운로드만 지원 (`GET /api/Files/{FileUri}`)
- **영향:** 그룹으로 업로드한 파일 N개를 각각 별도로 다운로드해야 함
- **해결 방향:** 프로젝트 요구사항에 따라 결정
  - 개별 다운로드 링크 N개 (현재)
  - ZIP 묶음 다운로드 (`GET /api/Files/group/{GroupId}`)
  - 둘 다 지원

### 11. 다운로드 Range 요청 지원
- **현재:** 스트리밍만 지원, 이어받기(Range) 미지원
- **개선:** S3 Range 요청 전달로 대용량 다운로드 이어받기 지원

### 12. 다운로드 중 취소 시 리소스 정리
- **현상:** 다운로드 요청 시 DB 레코드를 즉시 삭제하고 30분 후 스토리지 삭제를 스케줄함. 사용자가 다운로드를 중간에 취소하면 파일을 받지 못했지만 DB 레코드는 이미 삭제된 상태
- **영향:** 다운로드 취소한 사용자가 다시 다운로드할 방법이 없음
- **해결 방향:**
  - DB 삭제를 다운로드 완료 후로 지연 (Response 완료 이벤트 활용)
  - 또는 다운로드 횟수 제한 방식으로 변경 (즉시 삭제 대신 N회 허용)

### 13. 인증/인가 미구현
- **현상:** 모든 API 엔드포인트가 인증 없이 접근 가능
- **영향:**
  - 업로드: 누구나 파일 업로드 가능 → 스토리지 남용, 악성 파일 업로드
  - 다운로드: FileUri만 알면 누구나 다운로드 → 파일 유출
  - TusConfig: 서버 설정 노출
  - Hangfire Dashboard: Development 환경에서만 접근 제한되어 있으나 프로덕션 미설정
- **해결 방향:** NAS 배포 시 반드시 필요
  - JWT Bearer 또는 API Key 인증
  - tus 엔드포인트: `OnAuthorizeAsync` 이벤트에서 인증 검증
  - 다운로드: 토큰 기반 임시 링크 또는 인증 필수
  - Hangfire Dashboard: `IDashboardAuthorizationFilter` 구현
- **고려사항:**
  - NAS 환경에서 리버스 프록시(Nginx/Caddy) 레벨 인증도 가능
  - Blazor WASM은 별도 인증 플로우 필요 (cookie 또는 token)

### 14. tus 메타데이터 검증 시 불필요한 재시도
- **현상:** senderEmail/receiverEmail 누락 시 서버가 400 반환하지만 tus-js-client가 retryDelays에 따라 5회 재시도
- **원인:** tus-js-client가 400도 일시적 오류로 간주
- **영향:** 영구적 오류에 대해 불필요한 재시도 발생
- **비고:** 실제 UI에서는 클라이언트가 metadata를 항상 포함하므로 발생하지 않음. Console 직접 호출 시에만 발생
